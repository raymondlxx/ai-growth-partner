# 技术选型

## 语言与框架选型

### 后端

| 选项 | 语言/框架 | 适用场景 | 推荐度 |
|------|-----------|----------|--------|
| Spring Boot | Java/Kotlin | 企业级、稳定性要求高 | ⭐⭐⭐⭐⭐ |
| FastAPI | Python | AI服务、ML集成、快速迭代 | ⭐⭐⭐⭐ |
| NestJS | TypeScript | 前端团队转全栈、类型安全 | ⭐⭐⭐ |
| Go | Go | 高并发、微服务 | ⭐⭐⭐ |

**推荐**：
- 核心业务用 **Spring Boot (Java/Kotlin)** - 稳定、团队熟悉
- AI服务用 **FastAPI (Python)** - ML生态好、部署灵活
- 未来可考虑 **Go** 替换部分高并发模块

### 前端

| 选项 | 框架 | 适用场景 | 推荐度 |
|------|------|----------|--------|
| Next.js | React | SSR/SSG、需要SEO | ⭐⭐⭐⭐⭐ |
| Flutter | Dart | 跨平台App | ⭐⭐⭐⭐ |
| Taro/uni-app | React/Vue | 多端小程序 | ⭐⭐⭐⭐ |
| Vue 3 | Vue | H5/轻量场景 | ⭐⭐⭐ |

**推荐**：
- Web/管理后台用 **Next.js**
- 小程序用 **Taro**
- App用 **Flutter**

## 中间件选型

### 消息队列

| 选项 | 适用场景 | 推荐度 |
|------|----------|--------|
| Kafka | 高吞吐、日志、事件流 | ⭐⭐⭐⭐⭐ |

## 技术债务与风险

### 技术风险

| 风险 | 影响 | 应对 |
|------|------|------|
| 多系统集成复杂度高 | 高 | 先简化、逐步集成 |
| AI模型成本不可控 | 中 | 分层使用、优化Prompt |
| 数据隐私合规 | 高 | 早期咨询、加密存储 |
| 团队技术储备不足 | 中 | 技术培训、引入顾问 |

### 架构演进

```
阶段     架构                    说明
────────────────────────────────────────────────────
MVP     单体 + 核心微服务      快速验证、最小成本
中期     微服务拆分           按需拆分、避免过度
长期     平台化/中台化        能力复用、生态构建
```
