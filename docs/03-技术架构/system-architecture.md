## 技术栈

### 后端技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 语言 | Java 17+ / Kotlin | 稳定、团队熟悉 |
| 框架 | Spring Boot 3.x | 生态完善 |
| 微服务 | Spring Cloud | 服务治理成熟 |
| 数据库 | MySQL 8.0 | 事务数据 |
| 缓存 | Redis 7.x | 会话、热点数据 |
| 文档DB | MongoDB | 非结构化内容 |
| 分析库 | ClickHouse | 用户行为分析 |
| 向量DB | Milvus / Qdrant | 语义检索、Embedding |
| 消息队列 | Kafka | 异步、解耦 |

### 前端技术栈

| 端 | 技术选型 | 说明 |
|----|----------|------|
| App | Flutter / React Native | 跨平台 |
| Web | Next.js + React | SSR/SSG |
| 小程序 | Taro / uni-app | 一套多端 |
| H5 | Vue 3 / React | 轻量场景 |

| 能力 | 技术选型 | 说明 |
|------|----------|------|

| Embedding | text-embedding-3 / BGE | 语义向量化 |
| 情感计算 | 多模态模型 | 语音+文本 |
| 知识图谱 | Neo4j / 自研 | 技能关系 |
| RAG | LangChain / LlamaIndex | 知识检索增强 |

| 类别 | 技术 | 说明 |

## 服务划分

| 服务 | 职责 | 数据存储 |
|------|------|----------|
| user-service | 用户注册/登录/画像 | MySQL + Redis |
| content-service | 课程/内容管理 | MySQL + MongoDB |
| task-service | 任务系统核心逻辑 | MySQL + Redis |
| path-service | 路径规划引擎 | MySQL + 图数据库 |
| social-service | 社交/社群/聊天 | MySQL + MongoDB |
| payment-service | 支付/订单/会员 | MySQL |
| ai-profile-engine | 用户画像计算 | Redis + ClickHouse |
| ai-path-engine | 路径规划计算 | 图数据库 |
| ai-mentor-engine | AI导师对话 | 向量数据库 |
| aigc-service | AI内容生成 | 对象存储 |

## 核心设计原则

1. **微服务拆分** - 按业务域拆分，保持服务内聚
2. **事件驱动** - 异步通信通过Kafka，降低耦合
3. **读写分离** - 主从架构，读流量分摊
4. **缓存优先** - Redis缓存热点数据，减少DB压力
5. **AI能力下沉** - AI逻辑收敛到AI中台，统一服务
6. **弹性扩展** - K8s HPA，根据负载自动扩缩容
