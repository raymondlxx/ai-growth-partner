# 团队与资源

## 团队规划

### MVP阶段 (0-6月)

```
┌─────────────────────────────────────────────────────────────┐
│                     MVP团队配置 (5-7人)                      │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  产品 (1人)                                                  │
│  └── 产品经理 × 1                                           │
│      ├── 产品规划                                            │
│      ├── 需求管理                                            │
│      └── 项目跟进                                           │
│                                                              │
│  技术 (4人)                                                  │
│  ├── 后端工程师 × 2                                         │
│  │      Spring Boot, MySQL, 微服务                          │
│  │                                                          │
│  ├── 前端工程师 × 1                                         │
│  │      Next.js, React                                      │
│  │                                                          │
│  └── AI工程师 × 1                                           │
│          LLM集成, RAG, 对话系统                             │
│                                                              │
│  设计/运营 (1-2人)                                          │
│  ├── UI/UX设计师 × 1                                        │
│  │      产品设计, 交互                                        │
│  │                                                          │
│  └── 内容运营 × 1 (可选)                                    │
│          内容收集, 审核                                      │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 扩展阶段 (7-12月)

```
团队扩展至 12-18人

新增角色：
├── 产品经理 × 1 (B端)
├── 后端工程师 × 1-2
├── 前端工程师 × 1-2 (小程序/App)
├── AI工程师 × 1
├── 内容运营 × 1-2
├── 市场/增长 × 1-2
└── 客服/运营 × 1-2
```

### 生态阶段 (13-24月)

```
团队扩展至 20-35人

核心团队：
├── 产品/项目管理: 3-4人
├── 技术: 10-14人
├── AI/数据: 3-4人
├── 设计: 2人
├── 内容运营: 3-4人
├── 市场/增长: 3-4人
├── 运营/客服: 3-5人
└── 支持职能: 3-4人
```

## 核心岗位要求

### 产品经理

| 要求 | 说明 |
|------|------|
| 经验 | 3年+互联网产品经验 |
| 能力 | 需求分析、产品规划、项目管理 |
| 加分 | 教育/AI产品经验 |
| 性格 | 用户导向、数据驱动 |

### 后端工程师

| 要求 | 说明 |
|------|------|
| 经验 | 3年+ Java/Kotlin |
| 技能 | Spring Boot, MySQL, Redis, 微服务 |
| 加分 | 大规模系统、AI服务集成 |

### 前端工程师

| 要求 | 说明 |
|------|------|
| 经验 | 2年+ 前端开发 |
| 技能 | React/Vue, TypeScript, Next.js |
| 加分 | 跨平台、小程序开发经验 |

### AI工程师

| 要求 | 说明 |
|------|------|
| 经验 | 2年+ AI/ML经验 |
| 技能 | LLM, RAG, 向量数据库 |
| 加分 | 对话系统、Agent开发经验 |

## 资源需求

### 资金需求

| 阶段 | 资金需求 | 用途 |
|------|----------|------|
| MVP (0-6月) | 30-50万 | 团队、基础设施、AI API |
| 扩展 (7-12月) | 150-250万 | 团队扩展、市场推广、技术升级 |
| 生态 (13-24月) | 400-600万 | 规模化、生态建设 |

### 资金来源

| 来源 | 适用阶段 | 说明 |
|------|----------|------|
| 自筹 | MVP | 个人资金、亲友支持 |
| 天使投资 | MVP-扩展 | 种子轮 50-200万 |
| A轮融资 | 扩展 | 500-1500万 |
| 运营收入 | 生态 | 自造血 |

### 技术资源

| 资源 | 获取方式 | 说明 |
|------|----------|------|
| 云服务 | AWS/阿里云/腾讯云 | 按需扩展 |
| AI模型 | OpenAI/Anthropic/开源 | API + 自部署 |
| 工具链 | 开源+商业 | GitLab, Figma, Jira |
| 内容 | 自制+采购+共创 | 多渠道 |

### 人脉资源

| 资源 | 用途 |
|------|------|
| 行业专家 | 路径校准、内容顾问 |
| KOL | 种子用户、口碑传播 |
| 高校 | 课程合作、用户获取 |
| 企业 | B端合作、人才需求 |

## 团队建设

### 文化价值观

```
核心价值观：
────────────────────────────────────────────────────
1. 用户第一
   └── 一切决策以用户价值为导向

2. 快速迭代
   └── 小步快跑，快速试错

3. 开放透明
   └── 信息共享，坦诚沟通

4. 持续学习
   └── AI时代，团队和个人共同成长

5. 结果导向
   └── 以贡献和成果评价
```

### 协作机制

```
日常协作：
────────────────────────────────────────────────────
├── 每日站会 (15分钟) ── 同步进度、问题
├── 周会 (1小时) ─────── 复盘、计划
├── 双周迭代 ─────────── 敏捷开发
└── 月度复盘 ─────────── 目标对齐

沟通工具：
├── 飞书/钉钉 ─────────── 即时沟通
├── Jira ─────────────── 任务管理
├── Notion ───────────── 文档协作
└── GitLab ───────────── 代码协作
```

### 激励机制

| 激励类型 | 说明 |
|----------|------|
| 股权 | 核心成员期权池 |
| 现金 | 绩效奖金、项目奖金 |
| 成长 | 培训、书籍、学习时间 |
| 环境 | 弹性工作、远程办公 |
