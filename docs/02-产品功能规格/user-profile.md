# 全息用户画像系统

## 功能概述

通过多源数据融合，构建动态更新的用户画像，实现个性化的成长路径推荐和实时适配。

## 核心能力

### 1. 多维数据融合

| 数据类型 | 来源 | 用途 |
|----------|------|------|
| 基础属性 | 注册信息、问卷 | 人口统计、分群 |
| 学习行为 | 平台内操作日志 | 偏好识别、能力评估 |
| 生理数据 | 可穿戴设备API（选接） | 健康状态、精力管理 |
| 心理状态 | AI对话情绪分析 | 情感陪伴时机 |
| 生活事件 | 用户主动填报 | 路径调整触发 |
| 外部数据 | 职业背景（选填） | 职业转型匹配 |

### 2. 动态更新机制

```
用户画像 = 基础画像 + 行为增量 + 事件触发 + 时间衰减

触发场景：
├── 完成学习任务 → 能力提升
├── 生活事件变更（跳槽/育儿/搬家）→ 路径重匹配
├── 连续登录中断 → 激活提醒
└── 情绪低落 → 情感陪伴触发
```

### 3. 情感陪伴模块

| 功能 | 描述 | 触发条件 |
|------|------|----------|
| 深夜树洞 | AI倾听+情绪疏导 | 用户主动进入 or 连续3天低活跃 |
| 成长鼓励 | 正向反馈+里程碑庆祝 | 完成任务/达成阶段目标 |
| 困惑解答 | 快速响应+资源推荐 | 用户提问 or 频繁重复同一内容 |

## 数据模型

```
UserProfile
├── user_id: string
├── basic_info: { age, gender, education, occupation }
├── learning_data: { skill_levels, interests, preferences }
├── life_events: [{ event_type, timestamp, impact }]
├── emotional_state: { current_mood, trend[], triggers[] }
├── goals: [{ goal_type, target, deadline, progress }]
└── updated_at: timestamp

SkillLevel
├── skill_id: string
├── level: enum[入门/基础/进阶/精通]
├── confidence: float[0-1]
├── evidence: [task_completions[]]
└── last_assessed: timestamp
```

## 验收标准

- [ ] 用户完成初始画像问卷（15题，约5分钟）
- [ ] 画像系统能基于行为数据自动更新
- [ ] 支持手动触发生活事件更新
- [ ] 情感状态识别准确率（抽样）>70%
- [ ] 画像更新延迟 <1分钟

## 技术实现要点

- 初始问卷需精简，控制在5分钟内完成
- 画像存储采用时序数据库，支持历史追溯
- 情感分析使用轻量级模型，保护隐私
- 数据加密存储，用户可导出/删除
