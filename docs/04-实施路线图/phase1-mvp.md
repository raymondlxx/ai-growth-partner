# MVP阶段 (0-6个月)

## 阶段目标

**验证产品-市场契合（PMF）**，用最小成本验证核心假设。

## 成功标准

| 指标 | 目标 | 说明 |
|------|------|------|
| 日活用户 (DAU) | > 1000 | 验证用户愿意愿 |
| 次月留存率 | > 30% | 验证产品价值 |
| 任务完成率 | > 50% | 验证游戏化效果 |
| 用户满意度 | > 4/5 | 验证体验 |
| 付费转化率 | > 3% | 验证商业模式 |

### 不做 (Out of Scope)

- 社交功能（小组、排行榜）
- 情感计算引擎
- 知识图谱引擎

## 技术实现策略

### 快速启动方案

**前端**：Web优先，移动端H5

- 使用Next.js快速构建
- 先做Web，验证核心体验
- 小程序/Taro后期扩展

**后端**：单体架构 + 核心微服务

- Spring Boot单体，模块化设计
- 后续可按需拆分
- 数据库：MySQL（事务）+ Redis（缓存）

**AI能力**：优先使用成熟服务

- LLM：GPT-4 API（快速验证）
- 不自建模型，使用API封装
- 规则引擎 + 简单AI混合

### 技术债务控制

- MVP阶段允许一定技术债务
- 核心是验证业务，不是追求技术完美
- 但要保证代码可读、可维护
- 记录技术决策，便于后期重构

## 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 路径规划不够"智能" | 高 | 先用规则引擎，AI作为补充 |
| 用户完成率低 | 高 | 快速迭代，AB测试优化 |
| 留存率不达标 | 高 | 增加社交元素、情感陪伴 |
| AI成本超预算 | 中 | 优化Prompt、引入缓存 |
| 内容质量差 | 中 | 早期严格审核，用户反馈驱动 |
